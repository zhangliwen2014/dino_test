# 固定尺寸切块（Fixed-Size Tiling）设计文档

- 日期：2026-07-23
- 状态：已确认（用户 + 独立评审 agent-38 评审后修正）
- 取代：比例切块（`tile_grid` 网格，仍保留向后兼容）

## 1. 背景与目标

大图（如 2048×1536）上的小缺陷（10-15px 黑点/异色）在整图缩放到 224 输入时会被 patch（14px）稀释。切块（tiling）把原图切成小块分别提取特征，等效提升分辨率。

**决策：采用固定尺寸 T×T 切块，废弃比例切块（w/nx）。**

## 2. 决策理由（为什么废弃比例切块）

| 问题 | 比例切块（现行） | 固定尺寸 T（新方案） |
|---|---|---|
| 块尺寸 | 随图片尺寸浮动（w/nx），类别内图片尺寸不同时特征尺度不一致（PatchCore 硬约束无解，clamp_grid 只能缓解） | 恒为 T×T，尺度天然一致 |
| resize 畸变 | 非方形块强行 resize 成方形（2048×1536 下 x/y 缩放比 0.875 vs 1.167，畸变 33%） | 正方形块各向同性缩放，无畸变 |
| Batch | 块尺寸不一，无法统一拼批 | 全块 T×T，一次前向，可跨图拼批 |
| 代表图依赖 | auto 网格按首张训练图算 | T 为类别级常数，无需代表图 |

独立评审确认新方案更优，并修正了原提案 3 个漏洞（T 公式硬编码、小图尺度漏洞、长条图未定义）与 2 个建议（中点归属、1.5T 系数）。

## 3. 算法（修正版 v1.0）

### 3.1 块尺寸

```
T = P × (image_size / patch_size)
```

- P = 目标 patch 覆盖像素（原图），默认 20，可调（config `tile_target_patch_px`）
- image_size/patch_size：DINOv2 patch14 @224 → 16（T=320）；DINOv3 patch16 @224 → 14（T=280）
- **T 是类别级常数**，随版本 config.yaml 持久化（推理只读本版本的 T，不依赖当前全局配置）

### 3.2 滑动窗口布局

- `stride = T × (1 - overlap)`，overlap 默认 **0.15**（评审建议 ≥12.5%，边距 ≥ P）
- 起点序列：0, stride, 2×stride, …, 最后一块锚定贴边（起点 W-T），块尺寸恒为 T
- 每块都是 T×T 正方形，无畸形边缘块

### 3.3 边界规则

- **按轴独立**：轴长 ≥ T + 2P 才在该轴切分（保证该轴至少 2 块且各有 ≥1 patch 有效边距）
- **长条图**（短边 < T）：短边 pad 到 T（reflect 反射填充），保持块为正方形
- **小图**（两轴都 < T+2P）：整图各向同性放大到短边 ≥ T 后切（与记忆库尺度对齐）；评审备选：单独阈值/拒绝混训，v1 采用放大策略

### 3.4 拼接

- **重叠区中点归属**：相邻块中心连线的中垂线为归属线，每个像素恰好归属一个块（边距对称 = overlap×T/2 ≥ P）
- 单元测试覆盖：多种 W mod stride 余数下，每像素恰好归属一次

### 3.5 打分与批量

- 图片分数 = max（各块分数）；已知效应：块数随图面积变化 → max 统计量随尺寸漂移（Y4，记录在案暂不处理）
- 所有块统一 T×T → resize 到 image_size → **一次前向**

### 3.6 配置与兼容

- 版本 config.yaml 新增：`tile_scheme: "size" | "grid"`（缺省视为 "grid"）、`tile_size: T`、`tile_overlap`、`tile_target_patch_px`
- **旧 `grid` 版本不迁移**，按原逻辑正常推理（load_model_for_version 按 scheme 分流）
- **retrain 继承父版本 scheme**（再训练不改变切块方案，只有重新 train 才用新方案）
- 推理统一走 `score_one`（按版本 scheme 自动分流）

## 4. 旧方案的已知缺陷（保留 grid 仅为兼容）

- 块尺寸随图浮动 → 尺度不一致；非方形块 resize 畸变；max 块数固定掩盖了 Y4 效应但引入了密度钳制的尺度跳变

## 5. 验证标准

- 单元测试：T 公式（patch14/patch16）、滑窗起点序列、中点归属每像素恰好一次、边界规则、pad 逻辑
- 实测（blister 2048×1536）：未见 OK 图零误报；10-15px 黑点检出且红框标注；与旧 grid 版本（v008）对比 AUROC 与推理耗时
