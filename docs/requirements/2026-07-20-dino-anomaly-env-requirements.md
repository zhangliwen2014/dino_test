# DINO 无监督异常检测试验环境 — 需求文档

- 日期：2026-07-20
- 状态：待用户确认
- 对应设计文档：`docs/superpowers/specs/2026-07-20-dino-anomaly-env-design.md`

## 1. 背景与目标

构建一个基于 DINOv2/DINOv3 特征的**无监督工业异常检测试验环境**。核心任务：只用 OK（良品）图片训练，对测试图片自动判定 OK/NG 并输出异常热力图；当模型识别错误时，人工反馈真实标签，反馈累积后可增量再训练，生成改进的新版本模型。

典型场景：工业质检（先以 MVTec AD 公开数据集验证，后续接入自有图片）。

### 成功标准（验收条件）

1. 在 MVTec AD `bottle` 类别上完成全流程：训练 → 验证 → 测试 → 反馈 → 再训练 → 回滚，全部通过 CLI 和 Web UI 两种方式各走通一遍。
2. 基础模型在 `bottle` 测试集上图片级 AUROC ≥ 0.90（PatchCore 论文水平约 0.98，留有余量）。
3. 构造一个误报案例（NG 误判，实际 OK），反馈后再训练，该图判定翻转为 OK，且新版本整体 AUROC 不显著下降（下降 ≤ 2 个点）。
4. 环境在 CPU 上可运行（ViT-S 骨干），有 NVIDIA GPU 时自动使用 GPU。

## 2. 术语

| 术语 | 含义 |
|---|---|
| OK / good | 良品、正常图片 |
| NG / defect | 不良品、含缺陷图片 |
| 正常记忆库 | 训练时从 OK 图片提取的 patch 特征集合（coreset 采样后） |
| 缺陷原型库 | 从 NG 反馈图片提取的高分 patch 特征集合 |
| 误报 | 实际 OK 被判为 NG |
| 漏报 | 实际 NG 被判为 OK |
| 版本 | 一次训练/再训练的完整产物（checkpoint + 特征库 + 配置 + 指标） |

## 3. 功能需求

### FR-1 数据集管理

- **FR-1.1** 统一目录规范：`data/<类别>/train/good/`、`data/<类别>/test/good/`、`data/<类别>/test/<缺陷类型>/`，mask 可选（`mask/<缺陷类型>/`）。
- **FR-1.2** 支持下载 MVTec AD 并转换为上述规范（按类别选择下载）。
- **FR-1.3** 支持导入自有图片：指定类别名与标签（OK / NG+缺陷类型），自动归入规范目录。
- **FR-1.4** 数据集浏览：类别列表、各 split 图片数、缺陷类型统计、缩略图预览。
- CLI 与 Web UI（数据集页签）均支持上述能力。

### FR-2 训练（生成基础模型）

- **FR-2.1** 冻结骨干方式：预训练 DINOv2/DINOv3 提取 patch 特征，构建正常记忆库（PatchCore 方式，coreset 采样）。不做骨干反向传播训练。
- **FR-2.2** 骨干可配置：DINOv2 / DINOv3，模型大小（ViT-S/B/L），默认 DINOv2 ViT-S。
- **FR-2.3** 阈值自动校准：在 OK 验证图片上自动计算判定阈值；数据集缺 `test/good` 时自动从 `train/good` 切 20% 做校准并提示。
- **FR-2.4** 训练产物保存为新版本（见 FR-6）。
- CLI 与 Web UI（训练页签）均支持：选数据集类别、选骨干、调参数（coreset 采样率、输入尺寸，带默认值）、后台执行、显示进度与日志。

### FR-3 验证

- **FR-3.1** 全量验证：对指定模型版本跑整个 test 集，输出图片级 AUROC、AUPR、F1、阈值；有 mask 时输出像素级 AUROC/AUPRO。
- **FR-3.2** 选图验证：从数据集勾选或上传一张/多张图片，逐图显示分数、OK/NG 判定、热力图（不出聚合指标）。
- **FR-3.3** 验证结果逐图浏览，可按「只看误判」过滤（有 GT 时）；误判图可直接发起 OK/NG 反馈。
- **FR-3.4** 训练完成后自动执行一次全量验证；再训练后自动验证并与父版本指标并排对比。
- CLI 与 Web UI（验证页签）均支持。

### FR-4 测试

- **FR-4.1** 单图/批量推理：输出 OK/NG 判定、异常分数、异常热力图（缺陷定位）。
- **FR-4.2** 测试页支持上传图片或浏览数据集图片逐张测试。
- CLI 与 Web UI（测试与反馈页签）均支持。

### FR-5 反馈

- **FR-5.1** 对任意预测结果可标记「实际是 OK」或「实际是 NG」（可附带缺陷类型名）。
- **FR-5.2** 反馈记录持久化：图片（拷贝）、模型预测、人工标签、时间、模型版本。
- **FR-5.3** 反馈进入暂存区，**不影响当前模型**，在下一次再训练时生效（保证测试结果可复现）。
- **FR-5.4** 防污染护栏：OK 反馈图若原始异常分数 > 3 倍阈值，标记可疑并要求二次确认。

### FR-6 反馈再训练与版本管理

- **FR-6.1** 再训练 = 应用暂存区反馈，增量重建特征库：
  - OK 反馈 → 特征并入正常记忆库，重新 coreset 采样（库大小上限默认原库 1.5 倍）；
  - NG 反馈 → 高分 patch 特征并入缺陷原型库（全量保留）。
- **FR-6.2** 再训练生成新版本模型，版本化保存（checkpoint、特征库、配置、指标、父版本指针）。
- **FR-6.3** 再训练前显示预览（待并入 OK/NG 反馈数量、可疑项），需确认后执行；暂存区为空时拒绝执行。
- **FR-6.4** 支持版本列表、切换当前版本、回滚到任意历史版本；所有版本操作原子化，失败不破坏当前版本。
- **FR-6.5** 再训练后阈值自动重新校准，记录新旧阈值；新版本 AUROC 较父版本下降 > 2 个点时给出警告并提示可回滚。

### FR-7 双入口一致性

- Web UI 全部功能（数据集/训练/验证/测试/反馈/再训练/版本管理）与 CLI 一一对应，底层调用同一应用层 API。UI 为薄封装，不含业务逻辑。

## 4. 非功能需求

- **NFR-1 运行环境**：Windows；Python 环境用 venv + pip 管理。
- **NFR-2 设备自适应**：有 NVIDIA GPU 自动用 CUDA；无 GPU 用 CPU，可选 OpenVINO 加速推理。全流程 CPU 可跑通（ViT-S）。
- **NFR-3 可复现**：配置、版本、反馈全部落盘；同一版本对同一图片的判定结果可复现。
- **NFR-4 错误提示**：环境问题（权重缺失、CUDA 不可用、数据集结构不合规）在启动/操作早期发现，报错信息附带修复建议。
- **NFR-5 性能参考**：CPU + ViT-S 下，单图推理 ≤ 约 3 秒；bottle 类别（约 250 图）建库 ≤ 约 10 分钟。

## 5. 约束与依赖

- anomalib ≥ v2.5（DINOv3 骨干支持）；DINO 骨干经 timm 加载。
- DINOv2 权重免授权；**DINOv3 的 Meta 官方权重需 HuggingFace 授权（gated）**，timm 镜像权重是否免授权未确认——若需授权，环境需提供 HF token 引导。
- MVTec AD 数据集需下载（约 5GB，可按类别裁剪）。

## 6. 明确不做（Out of Scope）

- 骨干网络微调（全量/LoRA）——冻结骨干已满足需求。
- 在线标注工具、mask 编辑。
- 多用户、权限、服务端部署——本机试验环境。
- 实时视频流检测。
- OpenVINO INT8 量化调优（仅做导出可用即可）。

## 7. 风险与未决项

| 项 | 风险 | 应对 |
|---|---|---|
| DINOv3 权重授权 | gated 权重下载失败 | 提供 HF token 配置引导；默认 DINOv2 不受影响 |
| Patchcore + ViT 骨干层名兼容 | timm `features_only` 对 ViT 支持有限制 | 搭建早期做 spike 验证（集成冒烟测试覆盖） |
| Patchcore OpenVINO 导出 | 社区有导出 bug 报告（issue #2577） | OpenVINO 为可选增强，失败时回退 CPU PyTorch |
| anomalib API 变动 | 版本升级破坏子类化 | 锁定 anomalib 版本于依赖中 |
